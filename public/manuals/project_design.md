# 生态系统模拟器 · 项目规划
## 1. 项目简介
**1.1 目标**：构建一个纯前端的生态系统模拟网页应用。  
**1.2 核心功能**：用户可绘制食物网，设定物种参数及环境参数，运行模拟并观察种群动态、能量流动等变化。

## 3.核心功能
该项目核心计算模块用于模拟用户自定义食物网的种群动态变化，基于多猎物II型功能反应的营养动力学模型，同时集成了环境营养循环和基于代谢理论的自动参数生成。模拟以“天”为时间单位，采用一组耦合的常微分方程描述每个物种生物量以及环境无机营养池浓度随时间的变化。

1. 系统状态与变量

· 状态变量：每个物种的生物量 B_i（统一的可比生物量单位，如 gC/m² 或相对单位），以及环境无机营养浓度 N。
· 物种属性：分为生产者与消费者。每个物种具有体重 w_i（kg）、初始生物量 B_i(0)、内禀增长率 r_i（仅生产者）、自然死亡率 d_i、转化效率 e_i（仅消费者）、能量密度 \gamma_i（kJ/g）等。
· 捕食关系（连线）：每条连线记录了捕食者 j 对猎物 i 的攻击率 a_{ij}，以及捕食者 j 的处理时间 h_j（h 是捕食者生理属性，故同一捕食者的所有连线共享同一个值）。
· 全局环境参数：外源营养输入速率 I（营养单位/天）、分解速率 \varepsilon（死亡生物量矿化比例，0~1）、半饱和营养浓度 K_N，以及季节波动幅度、随机干扰频率等。

仿真时，所有参数可来源于用户手动设定，或由系统根据物种体重通过异速生长公式自动计算生成。

2. 核心动力学方程（由 formulas.js 中的函数实现）

2.1 生产者增长
生产者生物量变化率：

\frac{dB_i}{dt} = r_i B_i \cdot \frac{N}{K_N + N} - \sum_{j \in \text{捕食者}} \frac{a_{ij} B_i B_j}{1 + \sum_{k} a_{kj} h_j B_k}

· 第一项：遵循 Monod 营养限制的生长，当 N 趋于无穷时趋近最大增长率 r_i B_i。
· 第二项：被所有捕食者捕食的损失，分母是多猎物 II 型功能反应的特征和。求和遍历捕食者 j 的所有猎物 k，体现了各猎物对捕食者处理时间的分享。这使得不同猎物之间产生间接的似然竞争和稀释效应。

2.2 消费者增长
消费者生物量变化率：

\frac{dB_j}{dt} = B_j \left( e_j \cdot \frac{\sum_i a_{ij} B_i}{1 + \sum_k a_{kj} h_j B_k} - d_j \right)

· 第一项：所有猎物摄入率乘以转化效率 e_j，转化为自身生物量。
· 第二项：自然死亡损失。

2.3 无机营养池动力学
环境营养浓度 N 的动态：

\frac{dN}{dt} = I + \sum_{\text{所有物种}} \varepsilon d_i B_i + \sum_{\text{捕食关系}} (1 - e_j) \cdot \text{摄入量}_{ij} - \sum_{\text{生产者}} r_i B_i \cdot \frac{N}{K_N + N}

· I：外源输入。
· 第二项：所有物种自然死亡后，以比例 \varepsilon 快速矿化返还营养。
· 第三项：消费者摄入但未被同化的部分（排遗、排泄）直接进入营养池。
· 第四项：生产者吸收（与生产者方程生长项相同，确保质量守恒）。
  该模型将分解者功能隐含为全局营养循环，无需显式添加分解者节点，适用于湖泊、草原、森林等各类生态系统，仅通过 I, \varepsilon, K_N 调节生态场景。

3. 数值求解（由 solver.js 实现）

采用四阶龙格-库塔法（RK4） 进行时间积分，步长 \Delta t 通常设为 0.05~0.2 天以保证稳定性。每步计算过程：

1. 将当前状态（所有 B_i 和 N）作为输入，调用 computeDerivatives 函数，计算所有导数 dB_i/dt 和 dN/dt。
2. 根据 RK4 公式分四个阶段推进半步或整步，得到下一时间步的新生物量与营养浓度。
3. 可引入随机环境扰动（例如按概率施加丰度的瞬时减少）在步进后处理。

4. 辅助观测变量的实时计算

在每一步模拟后，引擎计算一系列不参与动力学的衍生变量，用于可视化：

· 个体数：N_i^{\text{ind}} = B_i / w_i（w_i 为成年个体平均体重）。
· 种群总储能：E_i = B_i \times \gamma_i（\gamma_i 为单位生物量能量密度，kJ/g）。
· 能量流动速率（猎物 i 到捕食者 j）：
  F_{ij} = \frac{a_{ij} B_i B_j}{1 + \sum_k a_{kj} h_j B_k} \times \gamma_i


  用于绘制能量流桑基图或连线粗细。
· 有效营养级：生产者固定为 1，消费者按食谱加权平均：
  \text{TL}_j = 1 + \frac{\sum_i \text{TL}_i \times \text{摄入量}_{ij}}{\sum_i \text{摄入量}_{ij}}


  用于绘制能量金字塔。

5. 基于体型的自动参数生成（autoParams.js 调用 formulas.js 中的异速方程）

用户仅需为每个物种输入成年个体体重 w（kg），系统自动推导其余参数：

· 内禀增长率（仅生产者）：r = r_0 \cdot w^{-0.25}，其中 r_0 由参考物种（如体重 1 kg 的兔子，r \approx 0.03 天⁻¹）校准。
· 自然死亡率：d = d_0 \cdot w^{-0.25}，常数 d_0 \approx 0.004 天⁻¹ 由参考物种校准。
· 攻击率（每条连线）：
  a = a_0 \cdot \frac{w_{\text{pred}}}{w_{\text{prey}}} \cdot e^{-\frac{w_{\text{pred}}/w_{\text{prey}}}{R_{\text{opt}}}}


  表现最优体长比 R_{\text{opt}}（默认 30）附近的捕食效率峰值；a_0 由参考捕食对（如青蛙 0.05 kg 吃蟋蟀 0.001 kg，a = 0.5）校准。
· 处理时间（捕食者属性）：
  h = h_0 \cdot w_{\text{pred}}^{-0.94} \cdot w_{\text{prey}}^{0.83}


  h_0 同样由参考对校准（例如 h_{\text{ref}} = 0.2 天）。
· 转化效率：默认 e = 0.1（10%定律），用户可调。
  所有自动生成的参数均可被用户手动覆盖，参数面板提供“自动/手动”切换。

6. 数据输入 / 输出

· 内部状态：物种数组、连线数组、环境参数均以 JavaScript 对象维护，符合特定的 JSON 结构。
· 导出：将当前食物网配置（物种参数、连线矩阵、环境设置）序列化为 JSON 文件，通过浏览器创建 Blob 并触发下载。
· 导入：用户上传 JSON 文件后解析并校验，恢复所有节点、连线和参数，同时可继续编辑或模拟。

7. 主循环与前后端协同

计算引擎完全运行在浏览器中，与 UI 解耦：

· React 组件通过状态管理（如 useState/useReducer 或 Zustand）持有编辑器数据和模拟运行时数据。
· 启动模拟时，将编辑器数据转换为 world.js 所需的紧凑结构（数组、邻接表），初始化 world 实例。
· 模拟运行时使用 requestAnimationFrame 驱动，每帧调用多次 solver.step() 累进模拟时间（如每帧推进 1 天），将结果推入历史数组供图表组件实时渲染。
· 暂停后可返回编辑页，系统用当前物种生物量覆盖初始值，用户可调整参数后继续计算，实现“自适应管理”的探索闭环。
